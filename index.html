What is AcmeWiki?

AcmeWiki is a modern, enterprise-grade wiki platform designed for teams that need structured, searchable, and permission-controlled documentation. Unlike traditional wikis that feel clunky or consumer tools that lack governance, AcmeWiki combines a rich editing experience with the access controls, audit trails, and deployment flexibility that organizations require.

Trusted by engineering teams, product orgs, compliance departments, and knowledge managers at companies of all sizes — from startups to Fortune 500 enterprises.

Core Features

Rich Block Editor

Every page is built from blocks — text, code, tables, images, embeds, databases, and more. Drag to reorder, nest, or convert between types. The editor supports real-time collaboration with live cursors, inline comments, and version history with diffs.

ℹ️
Real-time sync: All changes are saved instantly. Multiple users can edit the same page simultaneously — no merge conflicts, no "last save wins."

Role-Based Access Control

Granular permissions at the workspace, space, and page level. Define custom roles with read, edit, comment, or admin access.

Enterprise Search

Full-text search across all pages, PDFs, and attached documents. Supports boolean operators, filters by space/tag/author, and returns ranked results with highlighted excerpts. Semantic search powered by AI available on Business plans. 

// Search syntax examples

"api rate limits"        // exact phrase

space:engineering deploy*  // scoped + prefix

Getting Started Guide

1
Create your workspace

Choose a name and URL for your wiki. Invite your team members by email or SSO. Each workspace is isolated with its own data, settings, and permission structure.

$ npx acmewiki init my-org --region us-east
2
Set up spaces

Organize knowledge into spaces — think of them as top-level categories. Each space has its own sidebar, permissions, and page tree. Common spaces: Engineering, Product, HR & Ops, Sales Playbook.

3
Write your first page

Use / to open the block menu. Type headings, lists, code blocks, tables, or embed a Figma file. Use @ to mention a teammate or link another page.

4
Configure integrations

Connect Slack for notifications, Jira for issue linking, and GitHub for code references. Our webhook API supports custom integrations.

⚠️
Important: Workspace admins should review the security checklist before onboarding the first team. Ensure SSO and MFA are configured for production use.

API Reference

Interact with AcmeWiki programmatically via REST API. All endpoints return JSON and require authentication via API key or OAuth2.

GET /api/v1/pages 200 OK 
{
  "data": [
    {
      "id": "page_abc123",
      "title": "Welcome to AcmeWiki",
      "space": "engineering",
      "updated_at": "2026-06-12T14:30:00Z",
      "author": {
        "id": "usr_456",
        "name": "Sarah Chen"
      },
      "tags": ["onboarding", "getting-started"]
    }
  ],
  "pagination": {
    "cursor": "eyJvZmZzZXQiOjEwfQ==",
    "has_more": true
  }
}
💡
Use the cursor-based pagination pattern for large result sets. We return an opaque cursor string — pass it back as the ?cursor= parameter on the next call.
